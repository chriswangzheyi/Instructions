# **LLM**文档对话 **—— pdf**解析关键问题



## PDF 解析有哪些方法？区别是什么？

| 方法类别                            | 原理                                | 优点                                 | 缺点                       | 代表工具                                |
| ----------------------------------- | ----------------------------------- | ------------------------------------ | -------------------------- | --------------------------------------- |
| **基于文本流解析**                  | 解析 PDF 内部的文字流（Text Layer） | 速度快，结构保留较好                 | 对扫描件无效；复杂排版易乱 | pdfminer、PyPDF2、PDFPlumber            |
| **基于版面分析 (Layout Analysis)**  | 分析文字坐标、字体大小、位置关系    | 可还原段落、列、标题层级             | 坐标噪声高、易错位         | pdfplumber + layoutparser、pdfminer.six |
| **基于 OCR（光学字符识别）**        | 将 PDF 页转图片后识别文字           | 可处理扫描件、手写文档               | 精度受图像清晰度影响       | Tesseract、PaddleOCR、EasyOCR           |
| **基于深度学习版面分析 (LayoutLM)** | 模型学习文字+位置+视觉特征          | 可提取复杂结构（表格、标题、页眉等） | 训练成本高；推理慢         | LayoutLMv3、DocFormer、Donut            |
| **基于 AI 智能解析**                | 多模态 LLM 解析 PDF（视觉+语言）    | 自动抽取标题、摘要、表格、图像内容   | 成本高、可解释性弱         | GPT-4o、Qwen-VL、InternLM-XComposer     |



## 如何在长文档（书籍、报告）中提取关键信息？

针对“长文档”应采用 **分层结构化提取** + **语义摘要** 两步走：

##### ✅ Step 1：结构化抽取

- 按章节/标题层级切分；
- 保留页码、段落编号；
- 生成逻辑层级树（如 TOC → Section → Paragraph）。

##### ✅ Step 2：语义摘要

- 对每章节运行 LLM summarization；
- 保留关键概念、人物、指标；
- 用 Embedding 建立章节索引；
- 支持章节级检索与问答。

📘 实用技巧：

- 对超过 5000 tokens 的长章节：先摘要后索引；
- 对全书级任务（如主题挖掘）：使用 Map-Reduce Summarization。
